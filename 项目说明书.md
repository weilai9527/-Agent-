# 多 Agent 模拟面试与报告系统项目说明书

## 一、项目简介

本项目是一个面向求职者的模拟面试训练系统。系统通过多个面试 Agent 模拟技术面、架构面、HR 面等不同角色，帮助用户完成简历分析、面试配置、实时问答、单题评价、综合报告生成和能力统计。

项目采用前后端分离架构：

- 前端：React + Vite
- 后端：Python + FastAPI
- 数据库：MySQL

本地服务地址：

- 前端：http://127.0.0.1:5173/
- 后端：http://127.0.0.1:3001

## 三、核心功能

### 1. 用户账号模块

支持用户注册、登录、退出登录、会话保持和密码重置请求。登录状态通过 HTTP-only Cookie 保存，密码使用 scrypt 加盐哈希存储。

### 2. 个人资料模块

用户可以维护求职相关信息，包括昵称、目标岗位、经验水平、目标城市、期望薪资、技能标签、简历文本、项目经历、作品链接、默认面试类型、默认难度和面试官风格。

### 3. 简历分析模块

系统根据简历内容展示岗位匹配度、优势、风险点、可追问问题和优化建议，用于帮助用户完善面试素材。

### 4. 面试配置模块

用户可以选择目标岗位、经验级别、面试类型、公司场景、重点方向、面试强度和面试官风格，生成一场模拟面试。

### 5. 多 Agent 面试模块

系统支持创建多个面试 Agent，例如技术一面 Agent、架构二面 Agent、HR Agent。面试过程中会记录 Agent 提问、候选人回答、追问和系统消息。

### 6. 答案评价模块

用户提交回答后，系统会生成评分、优点、不足、建议和多维度评分，形成单题反馈。

### 7. 面试报告模块

面试结束后生成综合报告，包括总分、等级、录用建议、能力雷达、Agent 反馈、时间线复盘、总结和后续提升建议。

### 8. 历史记录模块

用户可以查看已生成报告的历史面试记录，并进入对应报告页。

### 9. 能力统计模块

系统基于历史报告累计统计用户面试次数、完成次数、平均分、技术深度、表达清晰度、业务理解等趋势数据。

## 四、技术架构

### 前端

- 使用 React 构建单页应用。
- 使用 Vite 作为开发服务器和构建工具。
- 本地开发时通过 Vite proxy 将 `/api` 转发到 FastAPI 后端。
- 通过 `fetch` 调用后端接口，并使用 `credentials: include` 携带 Cookie 登录态。

主要文件：

- `frontend/src/main.jsx`：页面、状态和接口调用逻辑。
- `frontend/src/styles.css`：整体视觉样式。
- `frontend/vite.config.js`：本地开发代理配置。
- `frontend/.env.example`：前端 API 代理和部署地址配置示例。

### 后端

- 使用 FastAPI 提供 REST API。
- 使用 PyMySQL 连接 MySQL。
- 启动时自动创建默认数据库、数据表、索引和缺失字段。
- 使用 HTTP-only Cookie 保存登录会话。
- 使用 scrypt 进行密码哈希，Token 只保存哈希值。
- 提供 OpenAI Realtime SDP 转发接口，用于实时语音面试场景。

主要文件：

- `backend/src/main.py`：FastAPI 应用、业务接口、面试流程和报告逻辑。
- `backend/src/database.py`：MySQL 连接配置、建库建表、索引创建和字段迁移。
- `backend/src/security.py`：邮箱规范化、密码哈希、密码校验、Token 生成和用户信息脱敏。
- `backend/src/env.py`：读取 `backend/.env` 环境变量。
- `backend/requirements.txt`：Python 后端依赖，包括 FastAPI、Uvicorn、httpx 和 PyMySQL。

### 数据库

数据库使用 MySQL。默认库名：

```text
multi_agent_interview
```

后端通过 `backend/.env` 配置连接信息：

```env
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your-mysql-password
MYSQL_DATABASE=multi_agent_interview
MYSQL_CHARSET=utf8mb4
```

运行前请确保 MySQL 服务可访问，并且配置用户具备创建数据库、创建表、创建索引和修改表字段的权限。

## 五、运行方式

### 1. 安装依赖

```bash
npm install
python3 -m pip install -r backend/requirements.txt
```

### 2. 配置后端环境变量

复制后端环境变量示例：

```bash
cp backend/.env.example backend/.env
```

然后根据本机 MySQL 修改：

```env
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your-mysql-password
MYSQL_DATABASE=multi_agent_interview
```

如需启用实时语音面试，还需要配置：

```env
OPENAI_API_KEY=your-api-key-here
OPENAI_REALTIME_MODEL=gpt-realtime-2
OPENAI_REALTIME_VOICE=marin
OPENAI_TRANSCRIPTION_MODEL=gpt-realtime-whisper
```

### 3. 启动后端 API

```bash
npm run dev:backend
```

后端默认运行在：

```text
http://127.0.0.1:3001
```

健康检查接口：

```text
GET /api/health
```

### 4. 启动前端页面

```bash
npm run dev:frontend
```

前端默认运行在：

```text
http://127.0.0.1:5173/
```

前端默认请求相对路径 `/api`，本地开发由 Vite 代理到 `http://127.0.0.1:3001`。如需部署到不同域名，可在 `frontend/.env` 中设置 `VITE_API_BASE_URL`，并在 `backend/.env` 中设置允许携带 Cookie 的 `FRONTEND_ORIGIN`。

### 5. 构建生产包

```bash
npm run build
```

### 6. 预览前端构建结果

```bash
npm run preview
```

## 六、主要接口

### 认证相关

- `GET /api/health`：健康检查
- `GET /api/auth/me`：获取当前登录用户
- `POST /api/auth/register`：注册
- `POST /api/auth/login`：登录
- `POST /api/auth/logout`：退出登录
- `POST /api/auth/password-reset/request`：请求密码重置

### 个人资料

- `GET /api/profile`：获取个人资料
- `PUT /api/profile`：更新个人资料

### 面试流程

- `POST /api/interviews`：创建面试
- `GET /api/interviews`：获取面试列表
- `GET /api/interviews/:id`：获取面试详情
- `PATCH /api/interviews/:id`：更新面试
- `POST /api/interviews/:id/start`：开始面试
- `POST /api/interviews/:id/finish`：结束面试

### Agent

- `GET /api/interviews/:id/agents`：获取面试 Agent
- `POST /api/interviews/:id/agents`：创建 Agent
- `PATCH /api/interviews/:id/agents/:agentId`：更新 Agent 状态

### 消息与评价

- `GET /api/interviews/:id/messages`：获取面试消息
- `POST /api/interviews/:id/messages`：新增消息
- `POST /api/interviews/:id/realtime/sdp`：创建实时语音 WebRTC 会话
- `POST /api/interviews/:id/evaluations`：创建回答评价
- `GET /api/interviews/:id/evaluations`：获取评价列表

### 报告与统计

- `POST /api/interviews/:id/report`：生成报告
- `GET /api/interviews/:id/report`：获取面试报告
- `GET /api/reports`：获取报告列表
- `GET /api/reports/:id`：获取指定报告
- `GET /api/stats/me`：获取个人统计
- `GET /api/stats/me/dimensions`：获取能力维度趋势

## 七、数据表设计概览

主要数据表包括：

- `users`：用户账号
- `profiles`：用户求职资料
- `sessions`：登录会话
- `password_reset_tokens`：密码重置 Token
- `interview_sessions`：面试会话
- `interview_agents`：面试 Agent
- `interview_messages`：面试消息
- `interview_evaluations`：单题评价
- `interview_reports`：综合报告
- `user_skill_stats`：用户能力统计

## 八、安全设计

- 密码不明文存储，使用 scrypt 加盐哈希。
- 登录态使用 HTTP-only Cookie，前端 JavaScript 无法直接读取。
- Session Token 入库前进行哈希处理。
- 后端所有个人资料、面试、消息、评价和报告接口都会校验当前用户身份。
- 登录失败会按 IP 与邮箱组合做基础频率限制。
- 跨域部署时通过 `FRONTEND_ORIGIN` 限制允许携带 Cookie 的前端来源。

## 九、项目特点

- 前后端分离，前端负责页面交互，后端负责业务接口和数据持久化。
- 后端采用 Python + FastAPI，适合面试展示和后续扩展。
- 数据库采用 MySQL，便于团队协作、部署和数据管理。
- 支持完整模拟面试闭环：配置、开始、问答、评价、报告、历史、统计。
- 支持多 Agent 面试官角色，便于模拟不同面试轮次和评价视角。
- 保留 OpenAI Realtime 接口扩展点，可接入实时语音面试能力。
