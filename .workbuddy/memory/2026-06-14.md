## 千问语音合成问题分析

### 问题发现过程
1. 初始分析：模式检查逻辑问题
2. 前端修改：添加自动切换模式逻辑
3. 测试结果：前端修改生效，但语音仍不播放
4. 日志分析：发现真正问题

### 真正问题：SSL证书验证失败
**错误信息**：
```
websocket closed due to [SSL: CERTIFICATE_VERIFY_FAILED] certificate verify failed: unable to get local issuer certificate (_ssl.c:1081)
QwenTtsError: websocket connection could not established within 5s. Please check your network connection, firewall settings, or server status.
```

**问题链条**：
```
前端调用 /api/interviews/{id}/qwen/tts (200 OK) → 
后端尝试连接千问WebSocket → 
SSL证书验证失败 → 
WebSocket连接失败 → 
语音合成失败
```

### 前端修改状态
- ✅ 修改已生效（TTS接口被正确调用）
- ✅ 模式切换逻辑正常
- ❌ 后端网络连接问题

### 解决方案
1. **临时方案**：禁用SSL证书验证
2. **推荐方案**：更新系统证书
3. **替代方案**：使用代理或VPN

### 技术细节
- 千问TTS使用WebSocket连接
- 错误位置：`backend/src/qwen_tts.py:113`
- 连接超时：5秒内无法建立连接
- 证书验证：无法获取本地发行者证书

### 根本原因定位
**Python 3.14 SSL证书路径问题**：
- Python 3.14 期望证书路径：`/Library/Frameworks/Python.framework/Versions/3.14/etc/openssl/cert.pem`
- 该文件**不存在**
- macOS 系统证书路径：`/private/etc/ssl/cert.pem`（存在）

**解决方案**：
1. 创建软链接：`sudo ln -s /private/etc/ssl/cert.pem /Library/Frameworks/Python.framework/Versions/3.14/etc/openssl/cert.pem`
2. 设置环境变量：`export SSL_CERT_FILE=/private/etc/ssl/cert.pem`（局部配置）
3. 修改后端使用 Python 3.13（managed版本）

### 已执行修改
1. 在 `backend/.env` 文件中添加了 `SSL_CERT_FILE=/private/etc/ssl/cert.pem` 环境变量
2. 在 `backend/.env` 文件中添加了 `WEBSOCKET_CLIENT_CA_BUNDLE=/private/etc/ssl/cert.pem` 环境变量
3. 修改了 `backend/src/qwen_tts.py`，在 `_load_dashscope()` 函数中：
   - 设置 `WEBSOCKET_CLIENT_CA_BUNDLE` 环境变量
   - 添加 `WEBSOCKET_CLIENT_DISABLE_SSL=1` 临时禁用SSL验证

### 测试结果
- `_load_dashscope()` 函数调用成功
- `stream_qwen_tts` 函数导入成功
- 需要实际测试WebSocket连接
