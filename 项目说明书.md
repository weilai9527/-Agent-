# 多 Agent 模拟面试与报告系统项目说明书

## 一、项目简介

本项目是一个面向求职者的模拟面试训练系统。系统通过多个面试 Agent 模拟技术面、架构面、HR 面等不同角色，帮助用户完成简历分析、面试配置、实时问答、单题评价、综合报告生成和能力统计。

项目采用前后端分离架构：

- 候选人端前端：React + Vite
- 候选人端后端：Python + FastAPI
- 管理端前端：React + Vite
- 管理端后端：Python + FastAPI
- 数据库：MySQL；管理端本地示例也支持 SQLite

本地服务地址：

- 候选人端前端：http://127.0.0.1:5173/
- 候选人端后端：http://127.0.0.1:3001
- 管理端前端：http://127.0.0.1:5174/
- 管理端后端：http://127.0.0.1:3002

## 三、核心功能

### 1. 用户账号模块

支持用户注册、登录、退出登录和会话保持。登录状态通过 HTTP-only Cookie 保存，密码使用 scrypt 加盐哈希存储。密码重置因当前没有邮件服务而暂缓启用。

### 2. 个人资料模块

用户可以维护求职相关信息，包括昵称、目标岗位、经验水平、目标城市、期望薪资、技能标签、简历文本、项目经历、作品链接、默认面试类型、默认难度和面试官风格。

### 3. 简历分析模块

系统根据简历内容展示岗位匹配度、优势、风险点、可追问问题和优化建议，用于帮助用户完善面试素材。

### 4. 职业能力目录与岗位治理模块

系统通过版本化数据维护“学院 → 专业 → 培养方向 → 就业方向 → 标准岗位 → 能力矩阵”，学院、专业和岗位均从数据库读取，不写死在代码中。计算机类数据作为首个试点，后续学院通过 Excel 批量导入草稿版本，经校验和审核后发布。

岗位需要有明确的数据来源，但不要求开发人员逐个修改代码添加。岗位维护遵循以下原则：

- 学院负责提供专业、培养方向、就业方向、标准岗位和初始能力要求。
- 管理端负责新增、编辑、停用、导入、审核和版本发布。
- AI 只提供岗位发现、去重、归类和能力矩阵补全建议，未经人工审核不得进入正式目录。
- 学生简历和面试配置只使用已发布目录中的标准岗位；模型临时生成的岗位不能直接写入正式数据。
- 目录变更通过新版本发布，不覆盖产生历史分析结果时使用的旧标准。

系统区分“标准岗位”和“企业招聘职位”。标准岗位相对稳定，用于学生能力评价和模拟面试；企业招聘职位带有公司、届次、地点和招聘时效，后续从招聘数据源同步，并映射到标准岗位。

推荐流程为：学院提供数据 → Excel 批量导入草稿 → 系统校验与 AI 辅助补全 → 专业负责人审核 → 管理员发布 → 学生端使用。

管理端左侧导航提供“职业目录”入口。目录页面展示当前版本的学院、专业、培养方向、就业方向和岗位树，并按数据权限提供新建草稿、Excel 导入和发布操作。正式版本只读，所有新增岗位先进入草稿版本。

简历推荐岗位采用“匹配与发现分离”的流程：

1. 先按当前已发布目录的正式岗位名称和岗位别名匹配。
2. 匹配成功时，把标准岗位 ID、目录版本和能力矩阵附加到简历分析结果。
3. 匹配失败时，在学生端明确标记为“目录外 AI 建议”，并按岗位名称聚合到管理端待审核池。
4. 管理员可以将建议合并为现有岗位别名、批准进入下一版草稿，或驳回。
5. “批准新增”不等于立即发布；岗位仍需补齐所属就业方向、编码和能力矩阵，并随新目录版本审核发布。

待审核池只保存岗位名称、累计出现次数、模型供应商、最近似标准岗位和审核信息，不保存学生简历正文或个人身份。相似度仅作为人工审核提示，系统不会根据模糊相似度自动并入正式岗位。相应数据表为 `catalog_job_aliases` 和 `catalog_job_suggestions`，管理入口位于“职业目录 → AI 岗位建议池”。

### 5. 面试配置模块

用户可以选择目标岗位、经验级别、面试类型、公司场景、重点方向、面试强度和面试官风格，生成一场模拟面试。

### 6. 多 Agent 面试模块

系统支持创建多个面试 Agent，例如技术一面 Agent、架构二面 Agent、HR Agent。面试过程中会记录 Agent 提问、候选人回答、追问和系统消息。

### 7. 答案评价模块

用户提交回答后，系统会生成评分、优点、不足、建议和多维度评分，形成单题反馈。

### 8. 面试报告模块

面试结束后生成综合报告，包括总分、等级、录用建议、能力雷达、Agent 反馈、时间线复盘、总结和后续提升建议。

### 9. 历史记录模块

用户可以查看已生成报告的历史面试记录，并进入对应报告页。

### 10. 能力统计模块

系统基于历史报告累计统计用户面试次数、完成次数、平均分、技术深度、表达清晰度、业务理解等趋势数据。

## 四、技术架构

### 前端

- 使用 React 构建单页应用。
- 使用 Vite 作为开发服务器和构建工具。
- 本地开发时通过 Vite proxy 将 `/api` 转发到 FastAPI 后端。
- 通过 `fetch` 调用后端接口，并使用 `credentials: include` 携带 Cookie 登录态。

主要文件：

- `frontend/src/main.jsx`：页面、状态和接口调用逻辑。
- `frontend/src/styles.css`：整体视觉样式。
- `frontend/vite.config.js`：本地开发代理配置。
- `frontend/.env.example`：前端 API 代理和部署地址配置示例。

### 后端

- 使用 FastAPI 提供 REST API。
- 使用 PyMySQL 连接 MySQL。
- 启动时自动创建默认数据库、数据表、索引和缺失字段。
- 使用 HTTP-only Cookie 保存登录会话。
- 使用 scrypt 进行密码哈希，Token 只保存哈希值。
- 提供 OpenAI Realtime SDP 转发接口，用于实时语音面试场景。
- 提供 Qwen-Omni-Realtime WebRTC SDP 代理接口，后端只做鉴权和信令交换，浏览器音频直连实时模型。
- 提供千问 CosyVoice 双向流式语音合成接口，用于文本驱动的面试官语音播放。

主要文件：

- `backend/src/main.py`：FastAPI 应用、业务接口、面试流程和报告逻辑。
- `backend/src/database.py`：MySQL 连接配置、建库建表、索引创建和字段迁移。
- `backend/src/security.py`：邮箱规范化、密码哈希、密码校验、Token 生成和用户信息脱敏。
- `backend/src/env.py`：读取 `backend/.env` 环境变量。
- `backend/requirements.txt`：Python 后端依赖，包括 FastAPI、Uvicorn、httpx、PyMySQL、DashScope 和 WebSocket 客户端。

### 数据库

数据库使用 MySQL。默认库名：

```text
multi_agent_interview
```

后端通过 `backend/.env` 配置连接信息：

```env
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your-mysql-password
MYSQL_DATABASE=multi_agent_interview
MYSQL_CHARSET=utf8mb4
```

运行前请确保 MySQL 服务可访问，并且配置用户具备创建数据库、创建表、创建索引和修改表字段的权限。

### 2. 配置后端环境变量

复制后端环境变量示例：

```bash
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env
cp admin_backend/.env.example admin_backend/.env
cp admin/.env.example admin/.env
```

然后根据本机 MySQL 修改：

```env
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your-mysql-password
MYSQL_DATABASE=multi_agent_interview
```

如需启用实时语音面试，还需要配置：

```env
OPENAI_API_KEY=your-api-key-here
OPENAI_REALTIME_MODEL=gpt-realtime-2
OPENAI_REALTIME_VOICE=marin
OPENAI_TRANSCRIPTION_MODEL=gpt-realtime-whisper
```

密码重置邮件能力当前暂缓，默认配置为：

```env
PASSWORD_RESET_ENABLED=false
VITE_PASSWORD_RESET_ENABLED=false
```

关闭后候选人端不展示“忘记密码”，相关接口不对外提供服务，生产环境也不依赖 SMTP 启动。已有实现保留，后续具备邮件服务时再开启功能并进行投递、过期和频率限制验收。

如需启用千问语音合成模式，还需要配置：

```env
DASHSCOPE_API_KEY=your-dashscope-api-key
DASHSCOPE_TTS_MODEL=cosyvoice-v3-flash
DASHSCOPE_TTS_VOICE=longanyang
DASHSCOPE_TTS_REGION=beijing
DASHSCOPE_WORKSPACE_ID=
```

如需启用 Qwen-Omni-Realtime WebRTC 直连模式，还需要配置：

```env
DASHSCOPE_API_KEY=your-dashscope-api-key
QWEN_OMNI_REALTIME_MODEL=qwen3.5-omni-plus-realtime
QWEN_OMNI_REALTIME_VOICE=Tina
QWEN_OMNI_REALTIME_WORKSPACE_ID=your-bailian-workspace-id
QWEN_OMNI_REALTIME_REGION=beijing
QWEN_OMNI_REALTIME_TRANSCRIPTION_MODEL=qwen3-asr-flash-realtime
QWEN_OMNI_REALTIME_TURN_DETECTION=semantic_vad
```

`QWEN_OMNI_REALTIME_WORKSPACE_ID` 未填写时会回退使用 `DASHSCOPE_WORKSPACE_ID`。后端根据地域自动生成官方 SDP 地址：北京使用 `{WorkspaceId}.cn-beijing.maas.aliyuncs.com`，新加坡使用 `{WorkspaceId}.ap-southeast-1.maas.aliyuncs.com`。只有代理或特殊部署场景才需要额外配置 `QWEN_OMNI_REALTIME_WEBRTC_ENDPOINT` 或完整的 `QWEN_OMNI_REALTIME_WEBRTC_URL`。

该模式下，浏览器通过 WebRTC 直接和 Qwen 实时模型传输音频；后端接口只负责校验登录态、拼接面试上下文、向 DashScope 交换 SDP answer，并返回 DataChannel 所需的 `session.update` 配置。

## 六、主要接口

### 认证相关

- `GET /api/health`：健康检查
- `GET /api/auth/me`：获取当前登录用户
- `POST /api/auth/register`：注册
- `POST /api/auth/login`：登录
- `POST /api/auth/logout`：退出登录
- `POST /api/auth/password-reset/request`：预留的密码重置请求（当前关闭）
- `POST /api/auth/password-reset/verify`：预留的一次性链接校验（当前关闭）
- `POST /api/auth/password-reset/confirm`：预留的确认改密接口（当前关闭）

### 个人资料

- `GET /api/profile`：获取个人资料
- `PUT /api/profile`：更新个人资料

### 面试流程

- `POST /api/interviews`：创建面试
- `GET /api/interviews`：获取面试列表
- `GET /api/interviews/:id`：获取面试详情
- `PATCH /api/interviews/:id`：更新面试
- `POST /api/interviews/:id/start`：开始面试
- `POST /api/interviews/:id/finish`：结束面试

### Agent

- `GET /api/interviews/:id/agents`：获取面试 Agent
- `POST /api/interviews/:id/agents`：创建 Agent
- `PATCH /api/interviews/:id/agents/:agentId`：更新 Agent 状态

### 消息与评价

- `GET /api/interviews/:id/messages`：获取面试消息
- `POST /api/interviews/:id/messages`：新增消息
- `POST /api/interviews/:id/realtime/sdp`：创建 OpenAI Realtime WebRTC 会话
- `POST /api/interviews/:id/qwen/omni-realtime/sdp`：创建 Qwen-Omni-Realtime WebRTC 直连会话
- `POST /api/interviews/:id/qwen/tts`：使用千问 CosyVoice 合成面试官语音
- `POST /api/interviews/:id/qwen/realtime-tts`：使用千问 CosyVoice WebSocket 流式合成面试官语音
- `POST /api/interviews/:id/evaluations`：创建回答评价
- `GET /api/interviews/:id/evaluations`：获取评价列表

### 报告与统计

- `POST /api/interviews/:id/report`：生成报告
- `GET /api/interviews/:id/report`：获取面试报告
- `GET /api/reports`：获取报告列表
- `GET /api/reports/:id`：获取指定报告
- `GET /api/stats/me`：获取个人统计
- `GET /api/stats/me/dimensions`：获取能力维度趋势

## 八、安全设计

- 密码不明文存储，使用 scrypt 加盐哈希。
- 登录态使用 HTTP-only Cookie，前端 JavaScript 无法直接读取。
- Session Token 入库前进行哈希处理。
- 后端所有个人资料、面试、消息、评价和报告接口都会校验当前用户身份。
- 登录失败会按 IP 与邮箱组合做基础频率限制。
- 跨域部署时通过 `FRONTEND_ORIGIN` 限制允许携带 Cookie 的前端来源。

## 九、项目特点

- 前后端分离，前端负责页面交互，后端负责业务接口和数据持久化。
- 后端采用 Python + FastAPI，适合面试展示和后续扩展。
- 数据库采用 MySQL，便于团队协作、部署和数据管理。
- 支持完整模拟面试闭环：配置、开始、问答、评价、报告、历史、统计。
- 支持多 Agent 面试官角色，便于模拟不同面试轮次和评价视角。
- 保留 OpenAI Realtime 接口扩展点，可接入实时语音面试能力。
- 支持 Qwen-Omni-Realtime 官方 WebRTC 链路，避免后端中转浏览器音频。
